# Port MCP

Use the `port-us` MCP server for Port-related work.

- Inspect available MCP tools and the relevant Port resources before acting.
- Prefer reading the current state before proposing or applying changes.
- Confirm the target blueprint, entity, environment, or scope before mutations.
- Explain the intended impact before destructive, bulk, or production-affecting operations.
- Do not invent Port resource identifiers or field names when the MCP server can provide them.
- Report the MCP result and any failed or skipped operations clearly.
